# SkyLuma AI Coding Guidelines

This file contains project-specific guidance for AI assisted development in SkyLuma.
It is not generated by the Angular CLI.

SkyLuma is a full-stack weather forecast application.

The project uses a monorepo structure:

- `backend/` contains the Spring Boot REST API.
- `frontend/` contains the Angular application.

## Backend guidelines

The backend uses:

- Java 21
- Spring Boot
- Maven
- YAML configuration
- Package root: `com.skyluma.weather`

Keep backend code organized by feature where possible.

Current main backend areas:

- `forecast/` for the public weather API
- `openweather/` for OpenWeather integration
- `config/` for application configuration
- `common/` for shared API/error handling code

Do not commit API keys, secrets, tokens, or local environment files.

Use `OPENWEATHER_API_KEY` for the OpenWeather API key.

Run backend tests with:

    cd backend
    ./mvnw test

## Frontend guidelines

The frontend uses:

- Angular
- TypeScript
- SCSS
- Angular routing

Keep feature code grouped under Angular feature areas.

Prefer:

- typed interfaces/models
- services for API communication
- components focused on presentation and user interaction
- SCSS files for styling

Run frontend build with:

    cd frontend
    npm run build

## General guidelines

Keep pull requests small and focused.

Do not mix unrelated backend and frontend changes unless the change is intentionally full-stack.

Prefer clear names over clever abstractions.

Avoid overengineering.

When changing the backend API contract, update the frontend models/services in the same PR only if needed.
